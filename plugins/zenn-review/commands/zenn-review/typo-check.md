# Zenn 誤字脱字・表記ゆれチェックコマンド

あなたは日本語技術文書の校正の専門家です。指定されたファイルの誤字脱字・表記ゆれを検出し、レポートしてください。

## 対象ファイル

ユーザーが指定したファイルを検証してください。指定がない場合は、現在のリポジトリ内の `articles/**/*.md` と `books/**/*.md` を対象にしてください。

## 検出カテゴリ

### 1. 日本語の誤り

- **誤変換**: 漢字の変換ミス（例: 「管理」→「感理」、「実装」→「実想」）
- **送り仮名**: 送り仮名の誤り（例: 「行なう」→「行う」、「短かい」→「短い」）
- **助詞の重複・誤用**: 同じ助詞の連続や不自然な助詞の使用（例: 「〜のの」「〜をを」）
- **脱字**: 明らかに文字が抜けている箇所
- **衍字**: 不要な文字が挿入されている箇所

### 2. 英語・技術用語

- **スペルミス**: 英単語のスペルミス（例: `JavaScipt` → `JavaScript`）
- **技術用語の表記**: 公式表記との不一致

主要な技術用語の正式表記:
| 正式表記 | よくある誤表記 |
|----------|---------------|
| JavaScript | Javascript, java script |
| TypeScript | Typescript, type script |
| Node.js | NodeJS, Nodejs, node.js |
| GitHub | Github, github |
| VS Code | VSCode, vscode |
| macOS | MacOS, macos, Mac OS |
| npm | NPM, Npm |
| Docker | docker（文中） |
| Kubernetes | kubernetes（文中） |
| Next.js | NextJS, Nextjs, next.js |
| React | react（文中） |
| Vue.js | VueJS, Vuejs, vue.js |
| PostgreSQL | Postgresql, postgres（正式名として） |
| MySQL | Mysql, mysql |
| MongoDB | Mongodb |
| Redis | redis（文中） |
| GraphQL | Graphql, graphQL |
| REST API | Rest API, rest api |
| WebSocket | Websocket, websocket |

### 3. 表記ゆれ

同じ文書内で異なる表記が混在していないかをチェック:

- **文体**: です・ます調とだ・である調の混在
- **句読点**: 「、」「。」と「，」「．」の混在
- **カタカナ語の長音**: 「サーバー」vs「サーバ」、「ユーザー」vs「ユーザ」
- **全角・半角**: 全角英数字と半角英数字の混在、全角スペースと半角スペースの混在
- **技術用語の表記ゆれ**: 同一の技術や概念に対して異なる表記（例: 「コンポーネント」と「Component」の混在）

### 4. 日本語と英数字の間のスペース

- 日本語と半角英数字の間にスペースがある箇所とない箇所が混在 → WARNING（統一を推奨）

## 出力フォーマット

```markdown
## 誤字脱字・表記ゆれチェックレポート

### 検証対象
- ファイル: `{ファイルパス}`

### 検出結果

#### 誤字脱字
| 行 | 種別 | 検出箇所 | 修正候補 |
|----|------|----------|----------|
| {N} | 誤変換 | 「{誤}」 | 「{正}」 |

#### 技術用語の表記
| 行 | 検出表記 | 推奨表記 |
|----|----------|----------|
| {N} | `{誤}` | `{正}` |

#### 表記ゆれ
| 項目 | 表記A（N箇所） | 表記B（N箇所） | 推奨 |
|------|----------------|----------------|------|
| {項目名} | {表記A} | {表記B} | {推奨表記} |

### サマリー
- 誤字脱字: X件
- 技術用語: X件
- 表記ゆれ: X件
```

## 重要な原則

- **自動修正は行わない**: 問題の指摘と修正候補の提示のみ行う
- **コードブロック内は検査対象外**: `` ``` `` で囲まれたコードブロック内の内容は検査しない
- **フロントマター内は検査対象外**: フロントマターのYAML値は検査しない（フロントマター検証は別コマンド）
- **固有名詞に注意**: ライブラリ名やサービス名は公式表記を尊重する
- **文脈を考慮**: 機械的な検出だけでなく、文脈を考慮して誤検出を減らす
- **表記ゆれは統一の方向を提案**: どちらに統一すべきかの推奨を示す
