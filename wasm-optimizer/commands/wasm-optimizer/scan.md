# wasm-optimizer: scan

プロジェクト全体の JS/TS ファイルをスキャンし、WASM 最適化の機会をレポートします。

## 手順

### 1. スキャン対象の確認

以下のディレクトリを走査対象とする:

- `src/`
- `lib/`
- `app/`
- `pages/`
- `components/`

以下は除外:

- `node_modules/`
- `dist/`, `build/`, `.next/`, `.nuxt/`
- `*.test.*`, `*.spec.*`（テストファイル）
- `*.config.*`（設定ファイル）

### 2. wasm-audit-reporter エージェント呼び出し

`wasm-audit-reporter` エージェントを呼び出し、プロジェクト全体の監査を実行する。

エージェントに以下の情報を渡す:

- プロジェクトルートパス
- `package.json` の依存関係情報
- バンドラーの種類

### 3. 結果の表示

エージェントの監査レポートをユーザーに表示する。

レポートには以下が含まれる:

- 既存 WASM 使用状況
- 最適化機会の優先度別サマリー
- 詳細一覧（ファイル、パターン、推奨ライブラリ、期待改善）
- 推奨アクション

### 4. 次のステップの提案

```
### 次のステップ

- 個別ファイルの詳細分析: `/wasm-optimizer:suggest [ファイルパス]`
- ライブラリ統合ガイド: `/wasm-optimizer:integrate [ライブラリ名]`
```
